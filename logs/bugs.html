Known Bugs in Ragnarok 01
=========================

* In the July 24 2023 build of the live iso, on some hardware (probably only
  on old machines), ttyS0 is respawning too fast causing the following error
  message:

  "init: Id "T0" respawning too fast: disabled for 5 minutes"

  This does not cause any real problems when booting in the default mode
  (live-x11), with the exception of the error message appearing when logging
  out of an X session. Although PS1 will disappear, typing startx then return
  will log back into X.

  This should be fixed in the next build by going from 115200 back to 9600.

* Exec bits were not set on the rcctl utility. Another bug that will be fixed
  in the next iso build. The temporary fix is to add them with the 'chmod 755'
  command if service management is needed.

* Firefox-esr crashes on first launch. It will start fine the second time
  however.

  This is most probably caused by hardened_malloc, as the crash does not occur
  when it is disabled. 

  This bug only affects firefox-esr, while the latest firefox provided by
  mozilla is not affected as it uses its own libraries.
