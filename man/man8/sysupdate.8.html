SYSUPDATE(8) System Manager's Manual SYSUPDATE(8)

NAME

sysupdate — manage base system binary patches

SYNOPSIS

sysupdate [-d | -h | -l | -q | -r]

DESCRIPTION

sysupdate is a small utility to fetch, verify and update Ragnarok's base system packages.

When run without any options, sysupdate will fetch the latest available updates for a base system program, then verify their signature with signify(1) before installing with the dpkg(1) command.

All operations, minus the dpkg(1) command, are performed as an unprivileged system user with no login shell access.

OPTIONS

The options are as follow:

-d
Download update without installing.
-h
Show help message.
-l
List installed updates.
-q
Check if there are any updates available.
-r
Revert an update.

FILES

/etc/sysupdate.conf
Configuration file containing the mirror path, Ragnarok's current version, and the path to the public signify key (/usr/share/ragnarok-keys/ragnarok01.pub).
/var/db/updates/sysupdate.list
File containing a list of currently installed updates.
/var/db/updates/sysupdate.list.old
When a system component gets updated, the old sysupdate list is backed up and can be used by the -r option to revert an update.

EXIT STATUS

The sysupdate utility exits 0 on success, and >0 if an error occurs. In particular, 1 indicates that the utility encountered and error, while 2 indicates that no installable updates were found.

SEE ALSO

signify(1), dpkg(1), sysupdate.conf(5)

AUTHORS

sysupdate was written by Ian LeCorbeau <I-LeCorbeau@protonmail.com>.

CAVEATS

sysupdate is only meant to update base system packages and does not resolve dependencies, as it assumes they are all satisfied.
