About Ragnarok

Ragnarok is a Debian derivative distribution currently under heavy development. The project's primary goal is to create a simple, Unix-like, and "hardened by default" distribution suitable as a daily driver. Still very much in its infancy, Ragnarok's immediate focus is to build a base system entirely from scratch, while remaining binary-compatible with the latest Debian stable release.

Guidelines

The project is developed while abiding to the following guidelines:

  • Be as independent as (realistically) possible from Debian.
  • Ensure the creation of a robust infrastructure which does not rely on Debian tools to build and distribute the base system.
  • Fetch sources from Debian whenever possible, but don't hesitate to fetch from upstream if necessary.
  • Document any changes made to Debian sources, including if/when a patch was reverted.
  • A lack of documentation should be considered a bug.
  • The base system should be treated as a whole and include all that is necessary to have a usable Unix-like system out-of-the-box, while remaining minimal.
  • Treat security as a first-class citizen and don't hesitate testing new security features in the development branch (-current).
  • Rebuild third-party packages (those from the Debian repository) if they cause any conflict with the base system.
  • Support for third-party packages which cannot (or will not) be made compatible with the base system should be removed swiftly.
  • All tools developed by the project should be usable either together, or on their own.
  • Decisions are made, and contributions accepted, on the basis of their technical merit alone.
